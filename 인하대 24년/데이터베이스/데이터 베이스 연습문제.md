# Ch1. 데이터베이스 개념
01. 데이터와 정보에 대한 설명으로 가장 적합한거. 
	- 정보는 데이터를 처리해서 얻을 수 있는 결과다.
02.  파일 처리 방식과 비교해 데이터베이스 관리 시스템을 활용할 때의 장단점에 대한 설명을 옳은거. 
	 - 데이터의 중복 최소화
	 - 데이터 일관성 유지
	 - 데이터 무결성 유지
03. 보기가 설명하는거.
	>관계형 데이터베이스 안의 테이블은 서로 연결해줄 수 있는 공통의 속성 공유. 이때 테이블 사이 같은 속성이 여러번 나타날 수 있는데 이를 설명하는 용어.
	>분산 데이터베이스에서 분할 투명성을 제공하기 위해 테이블의 투플이나 속성들을 여러 테이블로 나누어 중복 저장하는 것과 같은 의미.
	 - 중복 투명성
04. 데이터베이스의 정의로 가장 적합한거.
	- 공용 데이터, 통합 데이터, 저장 데이터, 운영 데이터
1. 데이터 베이스에 저장된 데이터의 특성
	- 운영 데이터, 통합 데이터, 종속 데이터
06. 데이터베이스 사용과 관련해 가장 옳은거.
	- 단일 사용자가 개인 혼자 사용하는 경우에도 데이터 베이스 사용이 효율적.
07. 데이터 베이스에 관한 설명.
	- 데이터베이스는 통합된 데이터다.
	- 효율적인 사용을 위해 실시간 처리가 가능해야 한다.
	- 데이터베이스 안의 데이터는 운영 데이터다.
08. OX 맞추기
	- 정보(Information)는 수집된 그대로의 정략적 혹은 정성적 실제 값을 말한다. (X)
	- 각 사용자 관점에 따라 정보가 또 다른 누군가에게는 데이터가 될 수 있다. (O)
	- 기업 전반의 운영 및 관리 정보를 통합 지원하는 정보 시스템은 전가적 자원 관리 시스템(ERP)다. (O)
	- 파일 처리의 근본적인 문제는 데이터 종속성과 데이터 중복성 문제다. (O)
	- 데이터 베이스의 주요 특성인 통합은 여러 장소에 흩어져 있더라고 상호 연결되어 접근할 수 있는 논리적 통합보다는 한 곳으로 모으는 물리적 통합을 뜻한다. (X)
09. 단어 채우기
	- (데이터)는 현실 세계에서 관찰 또는 수집의 결과로 나타난 사실이다.
	- 수집된 데이터를 저장했다가 필요할 때 제공하는 역할은 정보 시스템의 핵심인 (데이터 베이스)가 담당한다.
	- 기업들이 독립적인 부서를 통해 조직의 전체 데이터를 관리하게 하고 (CIO; Chef Information Officer)라는 최고 경영자를 임명해 의사결정에 필요한 정보를 관장하는 이유는 그만큼 기업 경쟁력의 핵심 요소이기 때문이다. 
10. 데이터와 정보의 차이를 구별해서 설명해라
	- 데이터는 현실 세계에서 관찰 또는 수집의 결과로 나타난 객관적 사실이다. 획득한 그대로의 정량적 혹은 정성적 실제 값이다.
	- 정보는 획득한 데이터에 의미를 부여하거나 가공  처리를 통해 의사 결정에 활용하도록 체계적으로 조직한 결과물이다.
11. 정보 시스템의 구조와 종류를 설명해라.
	- 정보(Information)을 데이터(Data)가 처리(Processing)하는 것을 함수로 I = P(D)로 표현할 수 있다. 
	- 종류로는 업무용으로
		- 생산 정보 시스템
		- 마케팅 정보 시스템
		- 인사 정보 시스템
		- 재무회계 정보 시스템
	기능용으로는
		- 거래처리 시스템
		- 경영정보 시스템
		- 전무가 시스템
		- 의사 결정 지원 시스템
		- 전사적 자원 관리 시스템
12. 파일 정보 시스템 방식과 비교할 때 데이터 베이스 시스템 방식이 갖는 장점은 뭐냐.
	- 데이터 종속성과 데이터 중복성, 데이터 보안과 동시성 제어가 가능해졌다.
13. 시대별 데이터 베이스 시스템의 성장 과정.
	- 수기 작성에서 컴퓨터의 등장, 데이터 베이스가 나오고 데이터 웨어하우스가 도입되고 빅데이터를 다루게 되었다.
14. 좁은 의미의 데이터베이스.
	- 데이터 베이스를 보다 효율적으로 다루기 위해 엄격한 제약을 갖는 데이터 저장소.
15. 데이터베이스의 주요 특성 4가지 설명. 
	- 실시간 접근(realtime accessibility): 데이터 베이스는 유저의 요구에 실시간으로 반응.
	- 끊임없는 변화(continous evolution): 끊임없이 동적으로 변화하는 데이터베이스.
	- 동시 공용(concurrent sharing): 여러 유저가 동시에 공용할 수 있어야 한다.
	- 내용 기반 참조(content based referencing): 데이터 참조를 위해 데이터의 저장 위치는 알 필요 없고, 찾고자 하는 데이터의 내용인 값만 알면 데이터 접근이 가능해야 한다.

# Ch2. 데이터 베이스 시스템
01. 데이터베이스 시스템의 장점
	- 데이터 통합성
	- 데이터 독립성
	- 데이터 일관성
02. 데이터베이스 관리자 (DBA)의 역할
	- 보안 및 권한 부여
	- 스키마 정의
	- 백업 및 회복
03. 데이터베이스 관리자(DBA)가 할 역할
	- 데이터 접근 권한과 회복 절차 수립 
	- 데이터베이스 구성요소 결정과 내부 저장 구조 정의 및 수정 
	- 시스템 감시 및 성능의 종합적인 분석과 성능의 개선
04. DBMS의 필수기능- 관련 데이터 언어-사용자 종류
	- 조작기능-(DML)-일반 유저
	- 정의기능-(DDL)-응용 프로그래머
	- 제어기능-(DCL)-데이터베이스 관리자
05. DDL과 관련 있는 단어들
	- - 데이터 사전(Data Dictionary): 데이터베이스의 구조, 제약조건, 테이블 및 속성에 대한 메타데이터를 저장하고 관리하는 시스템 테이블로, DDL 명령어로 데이터 사전에 정보를 정의한다.
	- 메타데이터(Metadata): 데이터에 대한 데이터로, 테이블, 열, 데이터 타입, 제약조건 등의 정보를 나타냅니다. DDL을 사용하여 메타데이터를 정의하고 관리한다.
	- 스키마(Schema): 데이터베이스의 구조를 정의하는 개념으로, 테이블, 뷰, 인덱스, 관계 등을 포함합니다. DDL을 통해 스키마를 정의하고 수정할 수 있다.
06. DBMS의 제어기능
	- 데이터의 무결성이 손상되지 않게 제어.
	- 데이터의 내용에 대한 정확성과 안전성을 유지할 수 있게 제어,
	- 여러 사용자가 데이터베이스에 동시 접근해 데이터 처리를 위한 병행 제어.
